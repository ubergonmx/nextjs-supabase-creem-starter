# Security Policy

## Supported Versions

| Version | Supported |
|---------|-----------|
| latest  | ✅        |

This is a boilerplate/starter template. Only the latest version on `main` receives security updates.

## Reporting a Vulnerability

**Please do not report security vulnerabilities through public GitHub issues.**

If you discover a security vulnerability, please report it by emailing:

> security@[your-domain].com

Include the following in your report:

- A description of the vulnerability
- Steps to reproduce the issue
- Potential impact
- Any suggested mitigations (optional)

## Response Time

- **Acknowledgement** — within 48 hours
- **Status update** — within 7 days
- **Fix / patch** — depends on severity; critical issues are prioritized

We appreciate responsible disclosure and will credit reporters in the changelog
(unless you prefer to remain anonymous).
